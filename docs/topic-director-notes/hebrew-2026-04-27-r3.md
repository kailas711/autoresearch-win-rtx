### Background context

HALOT (Koehler-Baumgartner Hebrew and Aramaic Lexicon of the Old Testament) often files high-frequency verbs across multiple lexical entries representing different stems or homographs. The pending-pile claim is:
- נתן (natan, "give") missing as a multi-stem entry
- שפט (shaphat, "judge") missing as a multi-stem entry

**Do not trust this claim.** The round 1 lesson is explicit: BDB binyan went from "1.1%" (pending-pile) to "86.2%" (actual fixture measurement) because the claim was never measured against the current fixture state. HALOT multi-stem may be at materially different coverage than implied. Re-measure first.

HALOT fixture field schema (confirmed by inspection of `lex_halot.json` first 3 entries):

```
aramaic_flag, binyanim, binyanim_source, bdb_page, cross_refs,
entry_health, entry_index, etymology, grammar, grammar_normalized,
grammar_normalized_source, headword, headword_consonantal,
headword_consonantal_source, headword_raw, headword_vocalized,
homograph_index, homograph_index_source, lexicon, parse_method,
primary_key, schema_version, senses, strongs, transliteration,
... (plus _source variants for most fields)
```

The `binyanim` field is present. The `grammar_normalized` field is the correct POS filter (not `grammar`). The `headword_consonantal` field is the correct headword lookup (not `lemma`).

**Domain knowledge note:** The HALOT-specific patterns section of `docs/domain-knowledge-cache.md` documents that HALOT uses `\n\t\t<lowercase-stem>:` markers for binyanim — distinct from BDB's spaced-capital-with-period format. The `parse_method` field may reflect how a given entry was classified (check if `parse_method` distinguishes single-stem vs multi-stem entries).

**Also relevant from domain-knowledge-cache.md §HALOT:** "If a verb appears absent from the multi-stem extraction (e.g., natan, shaphat reportedly missing): check if it's classified into a different category (CrossRef, single-stem) rather than truly absent. Source binary almost certainly has these as discrete entries."

### Acceptance script (runnable — adapt to actual schema as needed)

```python
import json, sys

# Stream-load to avoid full 22 MB in one shot if memory constrained;
# or load fully if memory is available — 22 MB is manageable.
with open('data/fixtures/lex_halot.json', encoding='utf-8') as f:
    data = json.load(f)

# --- Schema verification ---
print("Schema fields (first entry):", sorted(data[0].keys())[:25])
print("grammar_normalized values (sample 5):",
      [data[i].get('grammar_normalized') for i in range(5)])
print()

# --- Named samples: נתן (natan) and שפט (shaphat) ---
for lemma_cons in ['נתן', 'שפט']:
    matches = [e for e in data
               if e.get('headword_consonantal') == lemma_cons]
    print(f"{lemma_cons}: {len(matches)} entries")
    for i, e in enumerate(matches):
        bins = e.get('binyanim')
        print(f"  Entry {i}: grammar_normalized={e.get('grammar_normalized')}, "
              f"homograph_index={e.get('homograph_index')}, "
              f"parse_method={e.get('parse_method')}, "
              f"binyanim_count={len(bins) if bins else 0}, "
              f"binyanim={bins}")
print()

# --- Aggregate: verb entries and multi-stem coverage ---
# 'multi-stem' = at least 2 binyanim listed in the binyanim field
verbs = [e for e in data if e.get('grammar_normalized') == 'verb']
verbs_multistem = [e for e in verbs
                   if e.get('binyanim') and len(e['binyanim']) >= 2]
verbs_anystem = [e for e in verbs
                 if e.get('binyanim') and len(e['binyanim']) >= 1]
verbs_nostem = [e for e in verbs if not e.get('binyanim')]

print(f"Total verbs (grammar_normalized='verb'): {len(verbs)}")
print(f"  With >=1 binyan: {len(verbs_anystem)} "
      f"({100*len(verbs_anystem)/max(len(verbs),1):.1f}%)")
print(f"  With >=2 binyanim (multi-stem): {len(verbs_multistem)} "
      f"({100*len(verbs_multistem)/max(len(verbs),1):.1f}%)")
print(f"  With 0 binyanim: {len(verbs_nostem)}")
print()

# --- Bidirectional: over-extraction check ---
non_verb_multi = [e for e in data
                  if e.get('grammar_normalized') != 'verb'
                  and e.get('binyanim') and len(e['binyanim']) >= 2]
print(f"Non-verbs with >=2 binyanim (over-extraction): {len(non_verb_multi)}")
print()

# --- parse_method distribution for verbs ---
from collections import Counter
pm = Counter(e.get('parse_method') for e in verbs)
print(f"parse_method distribution (verbs): {dict(pm)}")
print()

# --- Are named samples present at all, even as single-stem? ---
for lemma_cons in ['נתן', 'שפט']:
    matches = [e for e in data
               if e.get('headword_consonantal') == lemma_cons
               and e.get('grammar_normalized') == 'verb']
    multi = [e for e in matches if e.get('binyanim') and len(e['binyanim']) >= 2]
    single = [e for e in matches if e.get('binyanim') and len(e['binyanim']) == 1]
    absent = [e for e in matches if not e.get('binyanim')]
    print(f"{lemma_cons} verb entries: {len(matches)} total "
          f"(multi={len(multi)}, single={len(single)}, absent={len(absent)})")
```
