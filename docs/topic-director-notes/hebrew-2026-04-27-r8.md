## 5. Phase-split decision

**Decision: Path B — 6-phase split.**

**Justification:**

The spec is 1,326 lines with 14 AC scripts spanning fundamentally different technical operations: HTTP fetch + XML parse (OSHB), TSV parse (STEPBible), treebank parse (Macula), hash computation (fingerprint), lexicon join algorithm (linguistic match), binary offset parsing (`.agloss`/`.ainter`), FK assembly (translations). Bundling into a single dispatch risks anti-pattern #2 (code-complete ≠ shipped: Builder implements all phases but never runs `build_corpus.py` end-to-end to produce the JSONL artifacts) and anti-pattern #3 (PASS conditional: "Phase 4 lexicon join deferred, rest is done"). Per `feedback_team_lead_practice.md` #5, single-defect-class dispatches are the right shape.

The 6 phases map cleanly to AC ownership and are sequentially dependent: each phase's output is the next phase's input. The coordination overhead of 6 cycles is lower than the cost of a bundled failure where 3 of 6 data artifacts are missing at Verifier time.

**Phase sequence:**

| Phase | Branch | Scope | ACs satisfied |
|---|---|---|---|
| 1 | `feat/plan-6-phase-1-ingest` | Fetch OSHB OSIS + STEPBible TSV; parse per-token surface/lemma/lemma_vocalized/morphology_code/root/paragraph_marker_post/cross_walk_references; produce raw token list (no token_id, no fingerprint yet) | Prerequisite for all |
| 2 | `feat/plan-6-phase-2-token-identity` | Assign token_id (Gen.1.1.0 format); compute BLAKE2b-128 fingerprint; produce `data/corpus/hebrew/v1.jsonl` v0 (no Macula, no joins, no glosses) | AC-1, AC-2, AC-6, AC-7, AC-8, AC-14 |
| 3 | `feat/plan-6-phase-3-passages` | Fetch Macula Hebrew; derive pericope boundaries; populate `phrase_role`/`phrase_depth`/`clause_role` on tokens; produce `data/corpus/hebrew/passages.jsonl` | AC-3 (gold verse coverage), AC-5 (passage FK), AC-9 (passage completeness) |
| 4 | `feat/plan-6-phase-4-lexicon-joins` | Execute linguistic-match join algorithm (§ F); populate `lexicon_links` on all tokens; update `v1.jsonl` | AC-4, AC-11 |
| 5 | `feat/plan-6-phase-5-glosses` | Implement STEPBible interlinear gloss join; optionally implement `.agloss`/`.ainter` parsers; populate `interlinear_glosses` on tokens; update `v1.jsonl` | AC-10 |
| 6 | `feat/plan-6-phase-6-translations` | Assemble `data/corpus/hebrew/translations.jsonl` from Berean (pericope-aggregated) + gold catalog; produce `manifest.json` | AC-12 |
| Final | (Verifier branch) | Run all 14 AC scripts on complete artifacts | AC-1 through AC-14 |

AC-13 (cross-walk coverage ≥85%) is validated by Phase 1/2 combined — the cross_walk_references field is populated in Phase 1 and fingerprint-exclusion is confirmed in Phase 2 (AC-14). The Verifier's final run confirms all 14 pass together.
